# CLI Reference (Source Notes)

> This file is **out-of-source documentation** about how the installed
> `reference.md` is produced. It is not read by the CLI at runtime.
>
> The installed `reference.md` skill file is **generated dynamically** from
> the clap command tree by `temper skill generate` / `temper skill install`.
> Both the command-table and the footer (resource types, stages, orientation
> projection, context-requirement matrix, etc.) live in
> [`src/commands/skill.rs`](../src/commands/skill.rs) as the
> `generate_reference()` function and the `REFERENCE_FOOTER` constant. Edit
> that file to change the installed reference.

## Invocation rule (preserved here for human reference)

**Always run `temper` directly from PATH.** Never use `cargo run -p
temper-cli`, `python`, full paths, or any indirect invocation method — even
when working inside the temper source repository. The installed binary may
differ from the in-development code, and that is intentional: we use the
installed CLI to manage our own workflow while evolving the crate.

Before running any temper command, verify the binary exists with `which
temper`. If it is not on PATH, install it with `cargo install --path
crates/temper-cli --locked` or ensure `~/.cargo/bin` is in your PATH.

## Skill-only commands

These are handled by the skill routing layer (the SKILL.md template), not the
`temper` CLI directly. They compose multiple CLI commands into guided
workflows:

| Skill Command | What It Does |
|---------------|-------------|
| `task start <slug>` | Shows task, moves to in-progress, routes to workflow |
| `task resume <slug>` | Shows task, reads last session, continues workflow |
| `task create` | Guided interactive task creation with prompts |
| `session start` | Start a session without a predefined task |
