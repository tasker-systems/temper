# Setup CI Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add GitHub Actions CI with code quality checks, testing, local git hooks, and a `test-embedder` feature flag to gate ML tests out of CI.

**Architecture:** Four reusable GitHub Actions workflow files (storyteller pattern: orchestrator calls code-quality + test-rust in parallel, ci-success gates branch protection). Local `githooks/pre-commit` mirrors the fmt+clippy checks. A `test-embedder` cargo feature replaces `#[ignore]` annotations on model-dependent tests.

**Tech Stack:** GitHub Actions, `dtolnay/rust-toolchain`, `Swatinem/rust-cache`, `cargo-audit`, Cargo feature flags

**Spec:** `docs/superpowers/specs/2026-03-23-setup-ci-design.md`

### Task 3: Create `code-quality.yml` reusable workflow

**Files:**
- Create: `.github/workflows/code-quality.yml`

- [ ] **Step 1: Create the workflow file**

```yaml
name: Code Quality

on:
  workflow_call:

jobs:
  quality:
    runs-on: ubuntu-latest
    timeout-minutes: 15

    steps:
      - name: Checkout code
        uses: actions/checkout@v4

      - name: Setup Rust
        uses: dtolnay/rust-toolchain@stable
        with:
          components: rustfmt, clippy

      - name: Setup Rust build cache
        uses: Swatinem/rust-cache@v2
        with:
          prefix-key: code-quality

      - name: Install cargo-audit
        run: cargo install cargo-audit --locked

      - name: Check formatting
        run: cargo fmt --all -- --check

      - name: Run Clippy lints
        run: cargo clippy --all-targets -- -D warnings

      - name: Check documentation builds
        run: cargo doc --no-deps --document-private-items
        env:
          RUSTDOCFLAGS: -D warnings

      - name: Security audit
        # Non-blocking — advisory database may have CVSS 4.0 issues
        continue-on-error: true
        run: cargo audit
```

- [ ] **Step 2: Commit**

```bash
git add .github/workflows/code-quality.yml
git commit -m "ci: add code-quality reusable workflow"
```

### Task 5: Create `ci.yml` orchestrator workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Create the workflow file**

```yaml
name: CI Pipeline

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

jobs:
  # ==========================================================================
  # Stage 1: Parallel quality + test jobs
  # ==========================================================================
  code-quality:
    uses: ./.github/workflows/code-quality.yml

  test-rust:
    uses: ./.github/workflows/test-rust.yml

  # ==========================================================================
  # Stage 2: CI Success Gate
  # All jobs must pass for CI to succeed (used as branch protection rule)
  # ==========================================================================
  ci-success:
    needs: [code-quality, test-rust]
    uses: ./.github/workflows/ci-success.yml
```
