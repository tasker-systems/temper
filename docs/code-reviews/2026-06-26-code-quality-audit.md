# Code-Quality Audit — Rust crates (2026-06-26)

Generated by the `code-quality-audit` workflow (`.claude/workflows/code-quality-audit.js`) against the
[code-quality best-practices lens](../development/code-quality-best-practices.md). Per-area read-only
auditors emitted rule-tagged findings; each area's findings were adversarially re-verified (false
positives and justified deviations dropped); the survivors were classified into the work-breakdown below.

- **Scope:** 16 crate-area units across the 9 Rust crates.
- **Confirmed findings:** 94 (after adversarial verification).
- **Run:** 32 agents, 512 tool calls, ~1.71M tokens.
- **Severity:** 6 high, 47 medium, 41 low.

> This report is the durable record. The PR-sized chunks in §3 are the backlog — each is
> independently reviewable and revertable; we deliberately avoid one mega-refactor PR.

## 1. Summary

The Rust workspace is structurally healthy but carries a long tail of consistency-debt that clusters into a few mechanical patterns plus a handful of genuine correctness risks. The dominant systemic patterns are CQ-7 hygiene (24 findings: missing public Debug, bare #[allow], undocumented magic constants), CQ-1 phase-bundling functions (17, concentrated in db_backend, substrate loaders/write, and CLI commands), and CQ-3 bare-Uuid params where typed newtypes already exist (14). Most findings are low-severity and mechanically fixable, but a small high-priority core matters: an unscoped public graph read (CQ-12 neighbors), two loose-marker-as-key reconcile hazards (CQ-2), two broken CLI tests (a tautological assertion and a mis-gated embed test), and a create/update validation pipeline already drifting across two db_backend methods. The systemic-vs-one-off split favors systemic: ~10 of 12 rules recur across ≥3 files, so the plan front-loads the few correctness fixes, then sweeps the mechanical patterns crate-by-crate to keep each PR independently reviewable.

## 3. Work breakdown (PR-sized chunks)

Ordered by priority. Each chunk is one cohesive, independently-shippable PR.

### 1. Scope or hide the unscoped public graph read (neighbors)

- **Priority:** high · **Effort:** M · **Rules:** CQ-12 · **Patterns:** unscoped-data-query
- **Files:** `crates/temper-substrate/src/readback/mod.rs`
- **Why:** Profile-scoping is a non-negotiable invariant; neighbors() is a pub fn returning kb_edges graph structure with no principal and no resources_visible_to join. Any current or future caller leaks cross-profile graph structure.
- **Scope:** IN: add principal: ProfileId + visibility join on both UNION ALL endpoints, OR demote to pub(crate) and document the outstanding access-model-gap task. OUT: the broader anchor_readable_by_profile context-edge gap (separate task) — this PR only stops the leak.

### 2. Key scenario edge-visibility checks on a stable edge id, not a label

- **Priority:** high · **Effort:** M · **Rules:** CQ-2 · **Patterns:** key-by-loose-marker
- **Files:** `crates/temper-substrate/src/scenario/access/runner.rs`, `crates/temper-substrate/src/scenario/access/loader.rs`, `crates/temper-substrate/src/scenario/access/model.rs`
- **Why:** kb_edges.label is a decorative non-unique TEXT column; fetch_optional on label silently returns a wrong row for same-label test edges, making EdgeVisibleTo checks pass/fail nondeterministically without erroring.
- **Scope:** IN: add key: String to AccessEdgeDef, capture the edge_id from fire(RelationshipAssert) into LoadedAccess.edges: HashMap<String,Uuid>, resolve through it in the runner. OUT: the readback KernelSliceRow doc fix (separate, different file/story).

### 3. Fix two broken CLI tests: embed gate and tautological assertion

- **Priority:** high · **Effort:** S · **Rules:** CQ-14 · **Patterns:** missing-test-embed-gate, tautological-assertion
- **Files:** `crates/temper-cli/src/actions/reconcile.rs`, `crates/temper-cli/src/actions/runtime.rs`
- **Why:** reconcile's embed test is gated on #[cfg(feature="embed")] (in default) so it runs in CI's no-ONNX jobs and fails at runtime — must be #[cfg(feature="test-embed")]. runtime's assert!(is_ok||is_err) is always true and gives zero coverage of the contract its name claims.
- **Scope:** IN: re-gate the reconcile test to test-embed; replace the tautology with assert!(matches!(result, Err(TemperError::Config(_)))). OUT: ingest test-name renames (cosmetic, separate).

### 4. Extract shared managed-meta validation pipeline from db_backend create/update

- **Priority:** high · **Effort:** M · **Rules:** CQ-1 · **Patterns:** phase-bundling-fn, duplicated-validation-pipeline
- **Files:** `crates/temper-api/src/backend/db_backend.rs`
- **Why:** create_resource (~122 lines) and update_resource (~135 lines) each inline the same strip→defaults→identity-keys→validate pipeline; a comment in update_resource literally says 'mirror create's pipeline'. The duplication is already a drift vector on every validation-rule change. Both are high severity.
- **Scope:** IN: extract resolve_caller_identity, build_managed_meta (strip+defaults+identity+validate), check_body_hash_dedup; both methods become short orchestrators sharing one validation helper. OUT: the bare-Uuid signatures in this file (separate ID-newtype PR).

### 5. Propagate serialization/serde errors instead of swallowing them

- **Priority:** high · **Effort:** S · **Rules:** CQ-6 · **Patterns:** swallowed-serialization-error, swallowed-serde-error

- **Scope:** IN: map_err+? on the two mcp sites and the one projection site. OUT: client authorize_url .expect() and core string-error (grouped into the library-error PR).

### 6. Correct KernelSliceRow diff-key documentation to resource_id
